## 시나리오 1: 기본 접근 + i18n 검증 (예상 5분)

### 1-1. KO 페이지 접근
1. https://tickdeck.site 접속 → 랜딩 페이지 로드 확인
2. https://tickdeck.site/app 접속 → 앱 페이지 200 OK
3. 아래 텍스트가 **한국어**로 표시되는지 확인:
   - 페이지 제목: "TickDeck — 당신의 웹사이트를 PPT로"
   - 히어로: "당신의 웹사이트를 PPT로"
   - 로그인: "시작하기" / "Google로 계속하기"
   - 폼: "홈페이지 주소", "회사명 (선택)", "슬라이드 생성하기"
   - 하단 가이드: "1 웹사이트 분석", "2 AI 슬라이드 제작", "3 PPT 다운로드"
   - 푸터: "💬 피드백", "☕ 후원"
4. `{{ t.` 같은 Jinja2 변수가 노출되지 않는지 확인
5. 영문이 섞여 나오지 않는지 확인

**통과 기준:** 모든 텍스트 한국어, Jinja 변수 미노출

### 1-2. EN 페이지 접근
1. https://tickdeck.site/en 접속 → EN 랜딩 로드
2. https://tickdeck.site/en/app 접속 → EN 앱 페이지
3. 아래 텍스트가 **영어**로 표시되는지 확인:
   - "Turn Your Website into a PPT"
   - "Get Started" / "Continue with Google"
   - "Website URL", "Company Name (optional)", "Generate Slides"
   - "💬 Feedback", "☕ Support"
4. **한글이 섞여 나오지 않는지** 확인 (특히 아래 항목 — 이전 QA에서 한글 잔존 발견됐던 곳):
   - 슬라이드 편집 모달: "Headline" (❌ "헤드라인" 아닌지), "Cancel" (❌ "취소"), "Save" (❌ "저장")
   - 에러 모달: "Please check the error details below." (❌ "아래 오류 내용을 확인해주세요")
   - AI 재생성 버튼: "🔄 AI Regenerate" (❌ "🔄 AI 재생성")

**통과 기준:** 편집/에러 모달 포함 전체 영문, 한글 미혼입

### 1-3. 언어 전환
1. KO /app에서 "🌐 EN" 클릭 → /en/app 이동 확인
2. EN /en/app에서 "🇰🇷 한국어" 클릭 → /app 이동 확인
3. 전환 후 UI 텍스트가 완전히 바뀌는지 확인

**통과 기준:** 양방향 전환 정상, 전환 후 텍스트 완전 교체

## 시나리오 3: 보안 검증 (예상 5분)

### 3-1. SSRF 차단 (S3)
1. 로그인 후 URL 입력란에 `http://localhost:6379` 입력 → 생성 클릭
2. **기대:** "내부 네트워크 URL은 사용할 수 없습니다" 에러 (400)
3. `http://192.168.1.1` 입력 → 동일 에러
4. `http://169.254.169.254` (메타데이터) → 동일 에러

### 3-2. Rate Limit (S4) ← 이전 QA에서 미작동이었음
1. 로그아웃 상태에서 `/api/auth/signup`에 POST 요청 6회 이상 반복
2. **기대:** 5회 후 429 Too Many Requests 반환
3. (개발자 도구 Network 탭 또는 curl로 확인)

```bash
# 터미널에서 테스트 (다른 이메일로)
for i in $(seq 1 7); do
  curl -s -o /dev/null -w "%{http_code} " \
    -X POST https://tickdeck.site/api/auth/signup \
    -H "Content-Type: application/json" \
    -d "{\"email\":\"test${i}@test.com\",\"password\":\"123456\"}"
done
```

**기대 결과:** 처음 5개는 200 또는 409, 6번째부터 429

### 3-3. Admin 접근 (S5)
1. **비관리자 계정**으로 로그인
2. `/admin` 접속
3. **기대:** /app으로 리다이렉트 (관리자 아님)
4. 관리자 계정으로 접속 시에만 대시보드 표시

### 3-4. OAuth CSRF (S1)
1. 개발자 도구에서 `/api/auth/google` 요청의 URL 파라미터 확인
2. **기대:** `state=` 파라미터 존재
3. 콜백 URL에도 `state=` 파라미터 포함 확인

### 3-5. 에러 메시지 (S6)
1. 존재하지 않는 URL로 생성 시도 (예: `https://thisdomaindoesnotexist12345.com`)
2. 에러 발생 시 상세 로그 토글 열기
3. **기대:** Python traceback, 파일 경로, API 키 등 시스템 정보 미노출
4. (이전 QA에서 `explicit_primary` 변수 에러가 노출됐었음 → 수정 완료)

**통과 기준:** SSRF 차단, Rate Limit 작동, Admin 비관리자 차단, state 파라미터 존재

## 시나리오 4: 슬라이드 생성 전체 플로우 (예상 10분)

### 4-1. 정상 생성
1. URL 입력: `martinee.co.kr` (마케팅 에이전시)
2. 회사명: 비워두기 (자동 감지 테스트)
3. 용도: "✦ AI 추천"
4. "슬라이드 생성하기" 클릭

**생성 중 확인:**
- [ ] 버튼이 "⏳ 생성 중..." 으로 변경됨 (D9 — 신규)
- [ ] 4단계 dot 순서대로 진행
- [ ] 진행 텍스트: "🔍 웹사이트를 분석하고 있어요" 등
- [ ] SSE 연결 끊김 시 지수 백오프 (3s→6s→12s) (M4)

**완료 후 확인:**
- [ ] "슬라이드 생성 완료!" 표시
- [ ] 슬라이드 미리보기 렌더링 정상
- [ ] 버튼 "슬라이드 생성하기"로 복원
- [ ] PPTX 다운로드 정상
- [ ] PDF 다운로드 정상

### 4-2. 슬라이드 품질 확인
다운로드한 PPTX를 열어서:
- [ ] **4, 5번 슬라이드가 다른 레이아웃** (연속 동일 방지)
- [ ] Contact(마지막) 슬라이드에 회사명 + 로고 표시
- [ ] CTA 슬라이드에 "X분 내", "X주 안에" 같은 시간 약속 없음 (A4)
- [ ] 헤드라인에 "5가지"라 했으면 body가 정확히 5개 (A1)
- [ ] 이미지 위 텍스트 가독성 OK (D16 — 오버레이 강화)
- [ ] 서브헤드라인이 바디보다 눈에 띄게 굵거나 큼 (D12)
- [ ] 카드 보더가 미세하고 세련됨 (D13 — 0.5px)

### 4-3. 슬라이드 편집
1. 미리보기에서 아무 슬라이드의 "편집" 버튼 클릭
2. 편집 모달 열림 확인
3. 헤드라인 텍스트 수정 → "저장" 클릭
4. 미리보기가 업데이트되는지 확인
5. "🔄 AI 재생성" 클릭 → 해당 슬라이드만 재작성

### 4-4. 토큰 환불 (실패 케이스)
1. 토큰 1개 남은 상태에서 **접근 불가한 URL** 입력 (예: `https://10.0.0.1`)
2. SSRF 차단으로 에러 발생
3. **기대:** 토큰 환불됨 (토큰 수 유지)

**통과 기준:** 생성 전체 플로우, 품질, 편집, 환불 모두 정상

## 시나리오 5: 모달 + 설문 (예상 5분)

### 5-1. 이력 모달
1. "이력" 버튼 클릭 → 모달 열림
2. 생성 이력 목록 표시 (방금 생성한 것 포함)
3. PDF 다운로드 버튼 표시
4. 모달 닫기 (X 버튼 또는 배경 클릭)

### 5-2. 피드백 모달
1. "💬 피드백" 버튼 클릭 → 모달 열림
2. 카테고리 선택 (🐛/💡/💬)
3. 텍스트 입력 후 "보내기" 클릭
4. "감사합니다!" 확인 메시지

### 5-3. 커피 후원 모달
1. "☕ 후원" 버튼 클릭 → QR 모달 열림
2. 카카오페이 QR 이미지 표시
3. 모달 닫기

### 5-4. 설문 (토큰 충전)
1. 설문 미참여 계정으로 로그인
2. "📋 설문 참여하고 토큰 2개 받기" 표시 확인
3. 설문 모달 열기 → Q1~Q9 작성
4. 제출 → "감사합니다! 토큰 2개가 지급되었습니다" 확인
5. 토큰 수 +2 증가 확인

**통과 기준:** 4개 모달 모두 정상 열림/닫힘, 설문 제출 + 토큰 지급
