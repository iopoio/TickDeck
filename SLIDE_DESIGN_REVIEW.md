# PPT 슬라이드 디자인 심층 리뷰 — 커버 / CTA / Contact

> 30년차 PD + 15년차 디자이너 관점
> 4개 디자인 스킬 (frontend-design, make-interfaces-feel-better, ui-ux-pro-max, web-design-guidelines) 기준

---

## 현재 상태 요약

지금 슬라이드가 "못생긴" 건 아니에요. 구조는 잘 잡혀있고, 다이나믹 컬러 시스템은 진짜 훌륭해요.
하지만 **"예쁜 템플릿"과 "세련된 프레젠테이션"의 차이**가 있어요. 그 차이는:

1. **여백의 자신감** — 지금은 요소들이 빽빽하게 배치됨
2. **타이포 위계의 극단화** — 헤드라인이 충분히 크지 않거나, 서브와 차이가 적음
3. **데코 요소의 절제** — 원, 바, 라인이 많아서 시선이 분산됨
4. **첫인상과 마지막 인상** — 커버와 마지막이 가장 오래 보이는데, 가장 generic함

## 1. 커버 슬라이드 (Cover)

### 현재 (이미지 없는 A-type 기준)

```
┌──────────────────────────────────┐
│█ 상단 primary 라인               │
│█                                 │
│█ COMPANY NAME (자간 넓게)      ○ │  ← 데코 원
│█                                 │
│█ 헤드라인                        │
│█ 여기에 큰 텍스트              ○ │  ← 데코 원 2
│█                                 │
│█ ── accent line ──               │
│█ 서브헤드라인                    │
│█ 로고                            │
│████████ 하단 primary bar █████████│
└──────────────────────────────────┘
```

### 문제점

**P1. 데코 요소 과잉 (높음)**
- 좌측 세로 bar + 상단 가로 라인 + 데코 원 2개 + accent line + 하단 bar = **6개 장식**
- 레퍼런스 PDF 분석에서도 대부분 1~2개 데코만 사용
- 너무 많으면 "무료 템플릿" 느낌이 남

**P2. 헤드라인이 충분히 크지 않음 (높음)**
- 현재 최대 64px. 슬라이드가 13.3"×7.5"인데 64px는 작아요
- 참고: Apple 키노트 커버 = 슬라이드 높이의 15~20% 차지
- 제안: 최대 80~90px, 최소 54px

**P3. 회사명과 헤드라인 사이 관계 모호 (중간)**
- 회사명이 11px 자간 넓게 + 헤드라인 64px → 위계는 있으나 "두 개의 텍스트 블록"으로 읽힘
- 회사명을 더 작고 미묘하게 (8px, transparency 40%) 하거나, 아예 eyebrow pill로 감싸면 관계가 명확

**P4. 좌측 정렬 획일적 (낮음)**
- A/B/C/D 전부 좌측 정렬. C-type이나 D-type은 중앙 정렬이 더 어울릴 수 있음

### 제안: A-type 커버 리디자인

```
┌──────────────────────────────────┐
│                                  │
│   [COMPANY NAME]  ← pill badge   │
│                                  │
│                                  │
│   헤드라인을                     │
│   여기에 크게                  ○ │  ← 데코 원 1개만
│                                  │
│   ── short accent ──             │
│   서브헤드라인 (muted color)     │
│                                  │
│                            logo  │
│ © 2026                    1 / 9  │
│█████████████████████████████████ │  ← 하단 bar만 유지
└──────────────────────────────────┘
```

변경 포인트:
- 좌측 세로 bar 제거 (하단 bar만 유지)
- 상단 가로 라인 제거
- 데코 원 1개만 (큰 것)
- 헤드라인 크기 UP (64→80px 범위)
- 회사명을 pill badge로 변경
- 전체적으로 여백 확대 (상단 padding 13%→18%)

## 2. CTA 슬라이드

### 현재 (A-type 기준)

```
┌──────────────────────────────────┐
│ ┌───────────┐  ○ 데코 원         │
│ │CTA SESSION│ pill                │
│ └───────────┘                    │
│                                  │
│   Unlock Your                    │
│   Growth with Us               ○ │
│                                  │
│ ┌────────────────────────────┐   │
│ │ sub: 컨설팅 한 문장        │   │  ← 이 박스가 어색
│ └────────────────────────────┘   │
│                                  │
│  · bullet 1                      │
│  · bullet 2                      │
│  · bullet 3                      │
│████████████████████████████████████│
└──────────────────────────────────┘
```

### 문제점

**P5. 구조가 커버와 너무 비슷 (높음)**
- 같은 다크 배경 + 같은 데코 원 + 같은 좌측 정렬 → 커버랑 거의 같아 보임
- CTA는 "행동 유도"가 목적이라 **시각적으로 전환점**이 되어야 해요

**P6. sub를 감싸는 roundRect 박스 (중간)**
- D-type에서 sub를 테두리 박스로 감싸는데, PPT에서 이런 UI 버튼 스타일은 어색해요
- 차라리 sub를 굵게 + 색상 강조하는 게 프레젠테이션답죠

**P7. body 불릿이 너무 작고 멀리 있음 (중간)**
- CTA의 body는 "다음 단계"를 설명하는 건데, 헤드라인과 거리가 멀고 폰트도 작아서 안 읽혀요
- CTA body는 카드형이나 아이콘 리스트로 표현하면 행동 유도력이 높아짐

### 제안: CTA 리디자인 방향

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│         CTA SESSION              │
│                                  │
│     Unlock Your Growth           │
│         with Us                  │  ← 중앙 정렬!
│                                  │
│   ─────────────────────          │
│                                  │
│  ┌─────┐  ┌─────┐  ┌─────┐     │
│  │ 01  │  │ 02  │  │ 03  │     │  ← 카드형 다음 단계
│  │분석 │  │전략 │  │실행 │     │
│  └─────┘  └─────┘  └─────┘     │
│                                  │
│████████████████████████████████████│
└──────────────────────────────────┘
```

변경 포인트:
- **중앙 정렬** (좌측 정렬 X) — CTA는 "선언"이라 중앙이 맞음
- 데코 원 제거 or 매우 미묘하게 (transparency 95%)
- body를 3-column 카드형으로 변환 (번호 + 설명)
- 또는 body 없이 헤드라인 + sub만으로 임팩트

## 3. Contact (마지막 슬라이드)

### 현재 (A-type 기준)

```
┌──────────────────────────────────┐
│█ ┌──────┐  ○                     │
│█ │CONTACT│                       │
│█ └──────┘                        │
│█                                 │
│█  회사명                         │
│█  (작게, 자간 넓게)            ○ │
│█                                 │
│█  헤드라인                       │
│█  (중간 크기)                    │
│█                                 │
│█  연락처 정보                    │
│█                                 │
│█ ─────── 구분선                  │
│████████████████████████████████████│
└──────────────────────────────────┘
```

### 문제점

**P8. 가장 중요한 페이지인데 가장 평범 (높음)**
- 이 페이지가 청중의 눈에 가장 오래 머무는데 (발표 끝나고 Q&A 동안 켜져있음)
- 현재는 그냥 "나머지 슬라이드랑 같은 느낌 + 텍스트만 좀 적은" 상태

**P9. 회사명이 너무 작음 (높음)**
- Contact에서 회사명/브랜드가 가장 크게 보여야 하는데, 현재 12~13px
- 참고: 레퍼런스 PDF에서 마지막 슬라이드는 회사명이 50~80px

**P10. 연락처 정보 배치가 밋밋 (중간)**
- body로 연락처를 나열하는데, 아이콘 없이 텍스트만 → 전화기인지 이메일인지 구분 안 됨

### 제안: Contact 리디자인 방향

```
┌──────────────────────────────────┐
│                                  │
│                                  │
│                                  │
│        ◆ COMPANY NAME ◆         │  ← 대형 (60~80px)
│                                  │
│        ─────────────             │
│                                  │
│        tagline here              │
│                                  │
│                                  │
│  📍 서울시 서초구    📧 info@    │  ← 아이콘 + 2열
│  📞 02-1234-5678    🌐 site.com │
│                                  │
│ © 2026 Company.              9/9 │
│██████████ gradient bar ███████████│
└──────────────────────────────────┘
```

변경 포인트:
- 회사명/로고를 **슬라이드 중앙에 대형 배치** (현재 좌상단 소형 → 중앙 대형)
- 연락처를 2열 그리드 + 아이콘 프리픽스
- 전체 중앙 정렬 (좌정렬 X)
- 배경을 커버와 동일 스타일로 (시작과 끝의 일관성)
- 하단 bar를 gradient로 (primary → accentLight)

## 전체 공통 개선 방향

### 여백 (Whitespace)
- 현재 상단 padding: H * 0.10~0.14 → **H * 0.16~0.20** 으로 확대
- 좌측 padding: W * 0.06~0.07 → **W * 0.08~0.10** 으로 확대
- "빈 공간이 무서워서 채우는" 패턴에서 벗어나기

### 데코 절제
- 모든 슬라이드에 데코 원 + 바 + 라인이 같이 나오면 "템플릿" 느낌
- 커버: 원 1개 + 하단 바
- CTA: 바 없이 배경색만
- Contact: 구분선 1개 + 하단 gradient bar

### 타이포 극단화
- 헤드라인: 현재 40~64px → **54~90px** (슬라이드 크기 대비 더 과감하게)
- 서브: 현재 16~20px → **14~16px** (헤드라인과 격차 벌리기)
- 이렇게 하면 "와 크다" → "아 이건 보조구나"가 한눈에 읽혀요
