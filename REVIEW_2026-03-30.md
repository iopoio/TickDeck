## 총평

TickDeck은 **URL → AI 분석 → PPT 자동 생성**이라는 명확한 가치 제안을 가진 Beta 서비스로,
멀티 에이전트 파이프라인(Researcher → Strategist → Copywriter → Image Matcher) 설계가 우수하고
6가지 내러티브 타입(A~F) 분류 체계는 업계에서 보기 드문 수준의 정교함을 갖추고 있다.

그러나 **보안 기초 공사**, **환각 방지 후처리**, **KO/EN 코드 중복** 등 핵심 영역에서
상용화 전 반드시 해결해야 할 이슈들이 존재한다.

## 잘된 점 (Strengths)

### 1. 멀티 에이전트 파이프라인 아키텍처
- Researcher → Strategist → Copywriter → Image Matcher 4단계 분리
- 각 에이전트가 명확한 책임과 입출력을 가짐
- Factbook 4000자 제한으로 토큰 효율 최적화

### 2. 6가지 내러티브 타입 분류 (A~F)
- Tech/SaaS(A), 제조(B), 크리에이티브(C), B2C(D), 공공(E), 개인(F)
- 업종별 슬라이드 구조가 다르게 설계됨
- C-type 엔터/아티스트 전용 처리 (showcase_work 중심)

### 3. 브랜드 색상 추출 정교함
- CSS 변수 → Elementor → 빈도 분석 → 파비콘 교차검증 5단계
- 외부 도메인 파비콘 스킵 (ZDNET 방지)
- 밝은 배경색 자동 반영 (luma 0.85~0.97)

### 4. 슬라이드 디자인 시스템
- 15종 이상 레이아웃 (cards, split, portfolio, numbered_process, stat_3col 등)
- 연속 동일 레이아웃 자동 방지 로테이션
- 글래스모피즘 카드, 대형 원형 데코, 아이브로우 필 등 모던 디자인

### 5. 견고한 인프라
- Flask + Celery + Redis 4-tier 아키텍처
- SSE 실시간 스트리밍 + 30회 재연결
- 토큰 시스템 + 실패 시 자동 환불

### 6. 프롬프트 가드레일
- RULE A~H 명시적 코드화 (환각 금지, 외부 텍스트 금지 등)
- CTA 시간/수치 약속 금지 (RULE F)
- 이미지 힌트 금지어 필터 (handshake, puzzle 등)
