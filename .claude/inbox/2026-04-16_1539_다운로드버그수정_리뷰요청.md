# 클과장 → 제대리 핸드오프: 다운로드 버그 수정 리뷰 요청

## 뭘 바꿨나

### 1. `backend/routers/slides.py`
- `PPTX_DIR = Path("tmp/pptx")` → `Path(__file__).parent.parent / "tmp" / "pptx"` (절대경로)
- 다운로드 엔드포인트 `download_pptx`에서 `authorization` 파라미터 제거 (auth-free)
  - 이유: 브라우저 `<a href download>` 태그는 Authorization 헤더를 못 보냄 → 항상 401 JSON 반환 → download.json으로 저장되던 버그

### 2. `backend/core/config.py`
- `env_file="../.env"` → `str(Path(__file__).parent.parent.parent / ".env")` (절대경로)
  - 이유: uvicorn을 `TickDeck/` 루트에서 실행 시 상대경로 `../` 가 엉뚱한 경로를 가리켜 Settings 로드 실패

## 왜 바꿨나
uvicorn을 `TickDeck/backend/`에서 실행하면 `from backend.core.config import settings` import가 실패(ModuleNotFoundError). `TickDeck/` 루트에서 실행해야 하는데, 그러면 기존 상대경로들이 다 틀어짐. 절대경로로 통일해서 어디서 실행해도 동작하게 수정.

## 실행 방법 변경
- 이전: `cd backend && uvicorn main:app --reload`
- 이후: `cd TickDeck && backend/.venv/Scripts/uvicorn backend.main:app --reload --port 8000`

## 검증 결과
- `/api/health` → 200 ✅
- `/api/slides/{id}/download` → 401 → **404** (인증 제거 확인, 기존 레코드는 상대경로 저장돼서 404 정상)
- 새 generation은 절대경로로 저장되므로 다운로드 정상 동작 예상 [미검증 - 실제 E2E 테스트 필요]

## 제대리 확인 요청 항목
1. `slides.py` PPTX_DIR 절대경로 계산이 올바른지 (`backend/routers/slides.py` 기준으로 `.parent.parent` = `backend/`)
2. `config.py` env_file 절대경로 계산이 올바른지 (`backend/core/config.py` 기준으로 `.parent.parent.parent` = `TickDeck/`)
3. 다운로드 엔드포인트 auth-free로 바꾼 게 보안상 문제없는지 (UUID = 48비트 랜덤, 추측 불가)
4. 새 generation E2E 테스트: URL 입력 → 생성 → 편집 → 확정 → PPTX 다운로드까지
