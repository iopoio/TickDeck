# TickDeck 차별화 포지션

## 핵심 원칙
TickDeck은 Gamma·Pitch·Tome·Canva·Slidesgo를 따라가지 않는다. 영문 글로벌 덱 시장은 레드오션. 한국어 + 공모전·제안서 + .pptx 편집 자유 영역으로 포지션 잡는다.

## 레드오션 (피할 것)
- Gamma·Pitch·Tome: 영문 글로벌 덱, 폐쇄 런타임. 이미 압도적 경쟁자
- Slidesgo·Canva 무료 템플릿: 가격·물량 경쟁 불가
- SlideModel 코퍼레이트 다이어그램: 개별 슬라이드 저작권 위험, 1:1 복제 금지

## 차별화 축 4개

### 1. 한국어 공모전·제안서 덱 특화 🟢 최상
- 후추님 본인이 2026년 공모전 6건 직접 경험 = 1차 데이터 자산
- Gamma/Pitch는 한국어 UX 거의 없음, 한국 공모전 양식(HWP 7p·BP 10p) 이해 0
- "심사위원 3초 룰 통과하는 표지 + 근거·수치 섹션 + 기대효과" 한국 공모전 문법 내장
- TAM: 매년 수만 건 공모전·사업계획서 작성자 (스타트업·공공·학생·1인 창업자)

### 2. URL 인풋 정교화 🟢 상
- Gamma는 프롬프트·키워드 위주. 긴 URL에 약함
- TickDeck은 블로그·GitHub README·뉴스 기사·논문·공공 공고 URL → 구조적 슬라이드
- 한국 기획자·개발자가 실제 다루는 URL 유형 우선 (네이버 블로그·브런치·티스토리·깃허브·공공 PDF)

### 3. 편집 자유도 (.pptx 네이티브) 🟢 상
- Gamma·Tome는 자체 런타임 안에서만 편집, export도 폐쇄 레이아웃
- TickDeck은 .pptx 파일 그대로 배출 → 후추님·사용자가 파워포인트·키노트에서 자유 후가공
- "AI로 70% 뽑고 30%는 내 손으로 다듬는다"가 한국 실무 니즈

### 4. 후추님 시그니처 스타일 프리셋 🟢 중
- 업체 템플릿 복제 X. 후추님 과거 PPT(USB+PC 아카이브)에서 직접 추출한 레이아웃·컬러·타이포 패턴
- "TickDeck 기본 스타일 = 후추님 감성"이 브랜드 DNA가 됨
- 추후 사용자별 시그니처 학습 기능으로 확장 가능

## 시드 추출 전략 (업체 복제 방지)

### ✅ 할 것
- 후추님 본인 PPT 아카이브(USB+PC) → .pptx 파싱 → 레이아웃·간격·컬러·폰트 JSON 추출
- Awwwards·Dribbble·Behance에서 "웹사이트·디자이너 실험작" 구조 원칙만 학습 (슬라이드 아님, 저작권 안전)
- 공공 자료·CC0·무료 라이선스 아이콘만 에셋으로 사용

### ❌ 안 할 것
- SlideModel·Envato·Slidesgo 템플릿의 도형·아이콘·레이아웃 1:1 복제 (상업 라이선스 위반)
- Gamma·Pitch·Tome 출력물 스타일 모사 (경쟁사 IP 회피)
- 유료 폰트 임베드 (Pretendard·Noto 등 오픈 라이선스만)

## 프리셋 설계 (3~5개 후보)

현 단계 후보 — 후추님 아카이브 분석 후 최종 확정:
1. "공모전 제안서" — 한국 공모전 HWP 양식 대응, 수치·근거 중심, 보수적 톤
2. "스타트업 피치" — 임팩트 타이포·다크 테마, 투자자 대상 (한국 VC 스타일)
3. "블로그 아티클 변환" — 에디토리얼 세리프, 긴 글 URL 입력 시
4. "리포트·분석" — 스위스 그리드·차트 중심 (Dribbble brutalist 계열)
5. "후추님 시그니처" — 본인 PPT 아카이브에서 가장 많이 쓰인 패턴

## 실행 타임라인

### Phase A — 시드 추출 (4/25 이후, AI챔피언 마감 후)
1. USB·PC PPT 아카이브 전수 스캔 (현재 진행 중)
2. 핵심 5~10장 선정 (후추님 승인) → python-pptx로 JSON 추출
3. 레이아웃·컬러·폰트 패턴 클러스터링
4. 프리셋 3~5개 초안 확정

### Phase B — 프리셋 구현 (5월 중)
1. TickDeck 생성 엔진에 프리셋 선택 옵션 추가
2. 각 프리셋별 .pptx 템플릿 마스터 생성
3. URL 인풋 유형별 프리셋 자동 추천 로직

### Phase C — 실전 검증 (5월 말 ~ 6월)
1. 공모전 제출 중인 본인 제출본을 TickDeck으로 재생성 → 품질 비교
2. 후추님 지인(작가·기획자) 10명 A/B 테스트
3. 피드백 수집 후 프리셋 보강
