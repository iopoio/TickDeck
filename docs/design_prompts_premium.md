## 🎯 TickDeck 컨텍스트

TickDeck는 **사용자 텍스트 → 자동 슬라이드** 도구. 평가 관점이 다름:
- 사용자가 입력한 텍스트는 절대 진리 (메시지)
- 자동 생성된 디자인은 도구 (메시지를 받치는가, 가리는가)
- 템플릿이 콘텐츠를 변형시키지 않는지 검증
- 빠진 슬라이드 / 합칠 슬라이드를 시스템이 자동 제안

**핵심 명제**: 디자인은 사용자 텍스트의 종이다. 텍스트가 더 강해지지 않는 디자인은 실패.

---

## 🌳 줄기 검증 (사용자 텍스트 분석)

```
사용자가 입력한 [N개 슬라이드 텍스트]를 보고:

1) 전체 줄기(한 줄 스토리)는 무엇인가? — 사용자 의도 추론
2) 감정 곡선: 도입/긴장/충격/해법/약속/마무리 어디인가
3) 줄기가 끊어지는 슬라이드 — 사용자에게 보충 슬라이드 제안
4) 줄기에 기여하지 않는 슬라이드 — 사용자에게 삭제/병합 제안

자동 생성된 디자인이 이 줄기를 받치는가, 가리는가?
```

---

## 🧭 콘텐츠 종속성 — 자동 생성 디자인 검증

```
자동 생성된 각 슬라이드에 대해:

1) 사용자 텍스트의 한 줄 메시지는?
2) 자동 생성된 디자인 요소(카드/박스/아이콘/배경/효과)가 그 메시지를
   어떻게 받치는가?
3) 받치지 않고 가리는 요소가 있는가?
4) 사용자 텍스트보다 디자인이 시각적으로 더 시끄러운가?
   → 그렇다면 디자인이 메시지를 잡아먹은 것. 톤 다운.
5) 더 단순한 표현(텍스트 한 줄, 숫자 하나)로 같은 메시지가 가능한가?

원칙: 사용자가 디자인을 보지 말고 메시지를 봐야 한다.
```

## ⚠️ TickDeck 자동화 특화 위험 신호

| 신호 | 진단 | 처방 |
|------|------|------|
| 모든 슬라이드가 같은 템플릿 | 단조로움 | 슬라이드 종류별 변형 |
| 텍스트 양과 무관하게 같은 사이즈 | 위계 부재 | 텍스트 길이로 음량 조절 |
| 자동 아이콘이 메시지와 무관 | 콘텐츠-형식 불일치 | 아이콘 매칭 검증 |
| 빈 영역을 자동으로 채움 | 빼기 사고 부재 | 빈 곳을 빈 채로 |
| 사용자 강조(굵게)가 무시됨 | 입력 신호 손실 | 사용자 의도 보존 |
